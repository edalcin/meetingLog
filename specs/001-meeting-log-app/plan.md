**Branch**: `main` | **Date**: 2026-03-14 | **Spec**: [spec.md](spec.md)

---

## Summary

Build a personal meeting log web application backed by MariaDB. A single Node.js (Hono) container serves both the REST API and a minimal Alpine.js + Tailwind CSS frontend. The app is protected by a PIN (env var), supports PWA installation, and is published to `ghcr.io/edalcin/meetinglog` via GitHub Actions on every push to `main`. Historical data from `memoriaReunioes-Reuniao.csv` is imported on initial setup via a one-shot script.

---

## Technical Context

**Language/Version**: Node.js 22 LTS
**Primary Dependencies**: Hono (web framework), mysql2 (MariaDB driver), Alpine.js CDN (frontend reactivity), Tailwind CSS CDN (styling)
**Storage**: MariaDB (external, managed independently); tables: `reuniao`, `schema_migrations`
**Testing**: Node.js built-in test runner (`node:test`) for unit tests; manual smoke tests for integration
**Target Platform**: Linux (Alpine) Docker container on UNRAID server
**Project Type**: Web application (monolithic single-container: API + static frontend)
**Performance Goals**: Filter response < 500ms; page load < 2s on LAN; Docker image ≤ 150MB compressed
**Constraints**: Single container, no build step for frontend (CDN-loaded), PIN auth via sessionStorage, no offline PWA caching
**Scale/Scope**: Single user, ~300–500 meeting records, LAN-only deployment

Alpine.js and Tailwind CSS are loaded from CDN — the Docker image contains only plain HTML/CSS/JS files in `public/`.

## Phase 0 Deliverables — Complete

See [research.md](research.md) for all technology decisions and rationale.

Key decisions:
- **Framework**: Hono (smallest Node.js footprint, modern API)
- **Frontend**: Vanilla HTML + Alpine.js CDN + Tailwind CDN (no build step)
- **Docker base**: `node:22-alpine` multi-stage build
- **Migrations**: `docker-entrypoint.sh` runs SQL before app starts
- **PWA**: manifest + minimal `sw.js` (install-only, no offline cache)
- **CI/CD**: `docker/build-push-action@v5` with GHA cache, `GITHUB_TOKEN`

## Implementation Milestones

> Detailed task decomposition is generated by `/speckit.tasks`.

### M1 — Database & Migration Foundation
- SQL schema (`001_init.sql`) for `reuniao` and `schema_migrations`
- `docker-entrypoint.sh` with DB wait + migration runner
- `import-csv.js` to import `memoriaReunioes-Reuniao.csv`

### M2 — Backend API
- `db.js` connection pool (mysql2, reads env vars)
- `routes/meetings.js`: GET list (filter/sort/paginate), GET :id, POST, PUT :id
- `server.js`: Hono app, mount routes, serve `public/` as static

### M3 — Frontend SPA
- `public/index.html`: PIN screen → meetings list view → new/edit form
- `public/assets/app.js`: Alpine.js component (PIN gate using sessionStorage, table with sort/filter, modal form)
- `public/assets/styles.css`: custom overrides
- Responsive layout (mobile-first, 375px+)

### M4 — PWA
- `public/manifest.json` (name, icons, start_url, display: standalone)
- `public/sw.js` (minimal registration-only service worker)
- iOS meta tags in `index.html` (apple-touch-icon, apple-mobile-web-app-capable)
- PWA icons (192×192, 512×512, 180×180 apple)

### M5 — Docker & CI/CD
- `Dockerfile` (multi-stage, node:22-alpine, ≤ 150MB target)
- `.dockerignore`
- `docker-compose.yml` for local dev
- `.github/workflows/docker-publish.yml` (push to main → GHCR)

### M6 — Documentation
- `docs/unraid-install.md` (UNRAID Docker Add UI instructions with env var table)
- `README.md` with brief overview and quick-start

## Risk Register

| Risk | Likelihood | Mitigation |
|------|-----------|------------|
| MariaDB unreachable at startup | Medium | `docker-entrypoint.sh` retries for 30s before failing |
| iOS PWA "Add to Home Screen" not discoverable | High (iOS limitation) | Include prominent in-app install instructions for iOS users |
| CSV import date parsing fails on edge rows | Low | `import-csv.js` logs and skips malformed rows; manual fix post-import |
| Docker image exceeds 150MB target | Low | Multi-stage build + Alpine + no frontend build artefacts keeps image lean |
